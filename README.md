# 🤖 AI Maturity Snapshot

A secure, interactive web application for assessing organizational AI maturity. Built with Vite, featuring a retro comic-style design and EmailJS integration for lead generation.

## 🚀 Features

- **5-Question Quick Assessment**: Rapid AI maturity evaluation across key organizational dimensions
- **Retro Comic-Style Design**: Bold colors, clear typography, and engaging visual elements
- **Instant Snapshot Generation**: Real-time results with visual progress bars and recommendations
- **Secure Email Integration**: Client-side EmailJS with honeypot spam protection
- **Responsive Design**: Works seamlessly on desktop, tablet, and mobile devices
- **Vite Build System**: Fast development and optimized production builds
- **GitHub Pages Deployment**: Automated deployment with GitHub Actions

## 📋 Prerequisites

## 🚀 Production Deployment

### Step 1: EmailJS Configuration

1. **Create EmailJS Account**
   - Go to [EmailJS.com](https://www.emailjs.com/)
   - Sign up for a free account
   - Verify your email address

2. **Add Email Service**
   - In EmailJS dashboard, go to "Email Services"
   - Click "Add New Service"
   - Choose your email provider (Gmail recommended)
   - Follow the authentication steps
   - Note down the **Service ID** (e.g., `service_abc123`)

3. **Create Email Template**
   - Go to "Email Templates"
   - Click "Create New Template"
   - Use this template structure:

```html
Subject: New AI Maturity Assessment Request - {{from_name}}

New AI Maturity Assessment Request

Contact Information:
- Name: {{from_name}}
- Email: {{from_email}}
- Company: {{company}}
- Title: {{title}}
- Phone: {{phone}}

Assessment Results:
- Maturity Level: {{maturity_level}}
- Total Score: {{total_score}}/20
- Percentage: {{percentage}}%

Key Recommendations:
{{recommendations}}

Additional Notes:
{{message}}

Detailed Quiz Answers:
{{quiz_answers}}

---
Generated by Fire Mountain Labs AI Maturity Assessment Tool
```

4. **Get Your User ID**
   - Go to "Account" → "API Keys"
   - Copy your **Public Key** (User ID)

### Step 2: GitHub Repository Setup

1. **Enable GitHub Pages**
   - Go to repository Settings
   - Scroll to "Pages" section
   - Set source to "GitHub Actions"

2. **Add GitHub Secrets**
   - Go to repository Settings → Secrets and variables → Actions
   - Add the following secrets:
     - `EMAILJS_USER_ID`: Your EmailJS Public Key
     - `EMAILJS_SERVICE_ID`: Your EmailJS Service ID
     - `EMAILJS_TEMPLATE_ID`: Your EmailJS Template ID

### Step 3: Deploy

1. **Build and deploy:**
   ```bash
   npm run build
   npm run deploy
   ```

2. **Or push to main branch for automatic deployment:**
   ```bash
   git add .
   git commit -m "Deploy AI Maturity Snapshot"
   git push origin main
   ```

3. **Monitor deployment:**
   - Go to repository Actions tab
   - Watch the "Deploy AI Maturity Snapshot" workflow
   - Check for any errors in the build process

4. **Access your site:**
   - Once deployed, your site will be available at:
   `https://yourusername.github.io/aim-quiz/`

## 🔧 Configuration Details

### EmailJS Template Variables

The application sends the following data to your email template:

- `{{from_name}}`: Contact's full name
- `{{from_email}}`: Contact's email address
- `{{company}}`: Contact's company name
- `{{title}}`: Contact's job title
- `{{phone}}`: Contact's phone number
- `{{message}}`: Additional notes from contact
- `{{maturity_level}}`: Calculated AI maturity level
- `{{total_score}}`: Total assessment score (0-20)
- `{{percentage}}`: Percentage score (0-100%)
- `{{recommendations}}`: Key recommendations (newline-separated)
- `{{quiz_answers}}`: Detailed quiz responses (JSON format)

### Security Features

- **No Hardcoded Secrets**: EmailJS credentials injected at build time
- **Honeypot Protection**: Hidden form field to catch automated spam
- **Client-Side Only**: No server-side processing required
- **Static Deployment**: Secure GitHub Pages hosting

## 🚨 Troubleshooting

### Common Issues

**EmailJS Not Working**
- Verify all three GitHub secrets are set correctly
- Check EmailJS dashboard for service status
- Test template in EmailJS dashboard

**Deployment Fails**
- Check GitHub Actions logs for errors
- Verify repository permissions
- Ensure all required files are present

**Quiz Not Loading**
- Verify `ai_maturity_quick_check.json` is in repository
- Check browser console for JavaScript errors
- Ensure JSON file is valid

**Build Errors**
- Check Node.js version (requires v16+)
- Clear node_modules and reinstall: `rm -rf node_modules && npm install`
- Verify all dependencies are installed

### Debug Mode

For local development with EmailJS:
1. Create `.env.local` file with your EmailJS credentials
2. Run `npm run dev`
3. Test email functionality locally
